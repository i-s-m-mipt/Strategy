1. Вести работу только в рамках директории others, другие части затрагивать только по согласованию со мной.
2. Для разработки новых компонент следует создавать отдельную ветку и по окончании работ вливать в мастер.
3. При работе в отдельной ветке, изменения из мастера подтягивать посредством команды rebase, а не merge.
4. Сообщения к коммитам писать на русском языке и достаточно развернуто, чтобы было понятно (50 символов).
5. Изменения в файле .gitignore можно вносить в корневой директории, но только по согласованию со мной.
